# 版本迭代记录

## V2.0 - DeepSeek 真实大语言模型 API 集成

### 本地代理服务
- 新建 `server/` 目录：Node.js + Express 本地代理服务
- `server/server.js`：DeepSeek Chat API 代理，`/api/analyze-care-request` 端点
- `server/.env.example`：API Key 配置模板（Key 仅存服务端，不暴露前端）
- `server/README_SERVER.md`：服务启动与配置说明
- 依赖：express、cors、dotenv、openai

### 前端 LLM 调用链
- `aiEngine.js` 新增 5 个函数：
  - `LLM_PROXY_URL`：代理地址常量 `http://localhost:3001`
  - `sanitizeRecordForLLM(record)`：PII 脱敏（姓名→"老人A"、手机号掩码、地址仅保留社区级）
  - `analyzeRequestWithLLM(record)`：fetch 调用代理，12 秒超时，失败返回 null
  - `buildFallbackAIResult(record, reason)`：规则引擎兜底包装
  - `analyzeRequestDefault(record)`：统一入口，LLM 优先 → 失败自动降级
- `app.js`：`submitCareRecord()` 改为 async，优先调 LLM
- 提交时 Toast："正在调用 DeepSeek 大语言模型进行风险识别..."
- LLM 成功/失败时分别 Toast 提示

### AI 来源可视化
- `ui.js`：AI 结果卡片顶部绿色/橙色来源标识条
- AI 元数据区：AI 来源、模型名称、是否真实 API、是否兜底、兜底原因、是否已脱敏
- `css/style.css`：新增 `.ai-source-banner`、`.ai-metadata` 等样式
- `renderAIDocPage()` 技术说明更新为 DeepSeek LLM

### 安全与隐私
- API Key 仅存 `server/.env`，`.gitignore` 排除
- 发送 LLM 前 PII 脱敏，不传输完整姓名/电话/门牌号/紧急联系人
- LLM 不可用时自动降级规则引擎，系统不崩溃
- 无 AI 模式切换 UI，默认 DeepSeek 优先
